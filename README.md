# 期货策略研究系统｜公开展示版

这是一个面向 **CTA / 期货量化策略研究岗位** 的脱敏作品集，用来展示我如何把主观交易观察转成可解释、可回测、可迭代、可交接的程序化策略。

项目的主线不是“搭一套大而全的平台”，而是解决策略研究中的实际问题：先筛选更值得交易的品种，再定义策略的方向、触发、退出和反手逻辑；利用因子证据与自动研究扩大实验覆盖，最后用因果回测和验证门槛决定候选能否晋级。

> 本仓库是从真实研究项目中抽取并重新实现的公开展示版。它不包含公司代码、私有数据、真实策略参数、实盘账户或可直接交易的 Alpha。

## 项目想解决什么

期货策略研究真正困难的不是计算 MA、ATR 或胜率，而是把下面的问题连成一个可复核的闭环：

1. **研究哪个品种**：当前行情是否有方向，趋势是否流畅，信息是否已经被价格消化？
2. **为什么交易**：方向、进场、退出和反手分别由什么结构触发？
3. **因子如何参与**：因子是提供样本证据和适用状态，还是在回测后拼凑解释？
4. **结果是否可信**：有没有未来函数，信号形成和执行是否错位，候选是否只适合某一段行情？
5. **怎样持续研究**：如何扩大实验数量，同时保存失败原因、控制样本外窥视并避免重复过拟合？
6. **怎样安全落地**：研究代码频繁变化时，如何避免更新直接影响实盘执行？

## 我的研究思路

```mermaid
flowchart LR
    A["市场信息与行情"] --> B["品种筛选"]
    B --> C["核心策略结构"]
    C --> D["因子证据与状态过滤"]
    D --> E["因果回测"]
    E --> F["验证与失败账本"]
    F --> G["候选策略晋级"]
    G --> H["冻结策略版本"]
    H -. "公开版不连接" .-> I["独立实盘执行"]
```

**策略设计是主线，因子和 AI 是研究加速器。**

- 核心技术指标先定义可解释的交易结构；
- 因子挖掘寻找策略在哪些市场状态下更有效；
- 自动研究扩大假设与组合的实验覆盖；
- 回测、验证集和封存样本负责最终裁判；
- 研究系统只产出冻结候选，不直接控制交易账户。

## 核心研究能力

下面使用三种标签说明公开范围：

- **公开代码实现**：可以在 `src/` 和测试中直接复核；
- **浏览器交互展示**：使用合成数据重现界面、研究口径和交互，不等于公开了原始计算服务；
- **目标工作流**：来自原始研究系统的方法和边界，本仓库只展示流程，不公开私有实现与数据。

### 1. 品种筛选：趋势流畅度与消息评分｜浏览器交互展示

趋势流畅度不是用一条抽象分数评价走势，而是统计价格对 **MA20 的有效穿越分段**：

- K 线实体连续两根位于 MA20 另一侧，才确认一次上穿或下穿；
- 穿越次数越少，说明趋势受到的反复干扰越少；
- 中位段幅和典型区间越大，说明一次趋势段提供的空间越充分；
- 支持 15 / 30 / 60 / 120 / 240 分钟、日线和周线；
- 区分窗口边界段与完整穿越段，避免首段统计口径含糊；
- 分别观察日盘以及不同夜盘收盘时间的品种组。

公开页面用确定性合成结果演示上述排序口径，没有开源原始系统的全品种扫描服务。消息评分页面同样用于表达一种研究方法：把趋势、供需、库存与基差、持仓变化、近期事件整理成带时间戳的输入，并判断信息是否已被价格消化。它用于品种筛选和策略过滤，不直接产生交易指令；本仓库不包含真实消息源和评分模型。

### 2. 策略研究：把交易思路变成可审计规则｜交互展示＋公开研究原语

策略研究室把自然语言交易思路拆成结构化定义：

- 决策周期与执行周期；
- 多空方向来源；
- 进场、退出和反手条件；
- 仓位与风险否决条件；
- 信号形成位置和实际执行位置。

在线页面演示“自然语言思路 → 结构化策略定义”的交互。本仓库分别公开了 AST 静态审计、信号合法性检查和逐根 K 线因果回测原语，但没有接入 LLM，也没有把页面按钮包装成自动生成、审计和执行的完整流水线。原始研究中，大模型用于提高表达与开发速度；未来函数、执行口径、样本划分和晋级标准仍由研究者明确约束。

### 3. 常规因果回测框架｜公开代码实现

每个信号包含：

- `signal_bar_idx`：信号在哪根已完成 K 线上形成；
- `exec_bar_idx`：可显式指定在哪根后续 K 线上执行；省略时由引擎默认到下一根；
- `action`：开多、平多、开空或平空；
- `reason`：产生信号的可解释原因。

默认在下一根 K 线执行。同 K 线执行、越界执行、重复开仓以及没有对应持仓的平仓会被拒绝，并保留结构化原因。这样既能快速测试普通策略，也不会因为向量化计算方便而模糊因果边界。

### 4. 30/60 分钟协同框架｜公开代码实现＋状态机示意

除常规策略框架外，项目还研究了“慢周期定方向、快周期找时机”的协同结构：

- 60 分钟已完成 K 线判断趋势环境；
- 30 分钟已完成 K 线寻找进入时机；
- 两个周期冲突时等待，而不是让快周期噪声推翻慢周期环境；
- 页面进一步展示了正常回撤、趋势弱化和明确反转的目标状态机；公开 Python 样例主要实现周期聚合、方向判断以及等待/开仓决策。

`build_effective_bars` 会过滤实体占整根振幅比例过低的低信息 K 线，减少小实体连续堆积对趋势结构判断的干扰，并在每根有效 K 线上保留原始 `source_index`，为后续把信号映射回真实时间轴提供依据。公开样例没有实现从有效 K 线生成信号再映射执行的完整流水线；这一点不能仅凭“删除 K 线”自动保证没有未来函数。

### 5. 支撑压力因子证据库｜浏览器交互展示

支撑压力因子模块不是画几条支撑线，而是把多周期状态诊断沉淀成可检索的证据目录：

- 研究类别：均值回归、中高周期趋势、低周期趋势；
- 因子结构：单状态、状态转折、跨周期组合；
- 查看方式：按品种查看因子，或按因子比较不同品种；
- 均值回归证据：方向一致率、剩余有利空间、最大不利波动、空间赔率；
- 趋势证据：未来 5 根路径胜率、强趋势胜率、退出盈利率和空间赔率；
- 原始系统的因子记录保留精确条件、统计口径、成功/失败样本和时间索引；公开页面使用少量合成卡片和说明文件展示这种信息结构。

公开仓库没有开源因子批量计算、样本数据库和真实导出服务；页面下载的是合成说明，用于展示报告格式。这里的“下一根开盘”只是统一的离线研究观察起点；方向一致率和路径胜率不等同于完整策略胜率。因子先回答“这种状态是否值得进入策略组合”，而不是直接包装成 Alpha。

### 6. 趋势因子策略组合：两条互不覆盖的研究路径｜目标工作流交互展示

策略组合不是给多个因子简单加权投票，而是分别研究 **进场、退出和反手结构**：

- **母策略演化**：在现有母策略上，每轮只增加、删除或替换一个结构，便于判断改动到底带来了什么；
- **独立策略研究**：从低、中、高周期趋势证据出发，独立形成一套完整策略，不覆盖母策略路线。

原始研究工作流会把候选与母策略或当前冠军比较交易覆盖、收益、回撤、月度稳定性、验证段改善和可实现空间赔率，通过保护门槛后再冻结策略版本并交给策略研究室回测。公开页面用合成候选、示意 JSON 路径和三行持仓 CSV 展示交接方式，并未实现后台自动生成或持久化这些制品。

### 7. 自动研究 Loop｜最小公开实现＋目标工作流展示

自动研究的目标不是让大模型自己宣布发现 Alpha，而是提高可证伪实验的吞吐量：

```text
提出假设 → 生成候选 → 代码审计 → 训练段研究 → 验证段判断
         → 一次性封存样本 → 晋级或失败账本
```

- 公开 Python 实现候选血统、验证门槛、一次性封存分数、晋级结果和失败账本；
- 页面展示更完整的目标工作流，包括不同研究角色、训练段与验证段分工、冠军记忆和失败记忆；
- 原始系统中的参数、样本范围、提示记忆和自动任务监督没有随本仓库公开；
- 核心纪律是只有验证达标才能查看封存结果，“本轮相对最好”不等于自动晋级。

### 8. Local GEX 与趋势研究｜浏览器可视化原型

Gamma 趋势实验室观察现价附近执行价的 Local GEX 结构，用来判断期权对冲行为更可能抑制波动还是放大已有趋势。

- 趋势方向仍由期货价格结构给出；
- Local GEX 只用于判断趋势延续环境，不单独生成多空信号；
- 负 GEX 被视为可能放大已有方向的条件，而不是“必然出现大趋势”；
- 公开版采用代理聚合口径，不代表真实做市商净仓位。

仓库中的柱状分布是固定合成数据，用于说明 Local GEX 的解读和策略边界；没有开源真实期权链清洗、Gamma 计算或持仓聚合服务。

### 9. 研究与执行隔离｜目标架构边界

研究系统需要频繁调整策略、因子和页面，因此不应该直接连接实盘账户。目标架构中，研究端只输出经过验证并冻结的策略版本；独立执行服务再负责行情、订单、持仓对账和异常恢复。

独立执行服务不在本仓库。本仓库只展示研究端和模拟执行状态，不包含账户配置、柜台连接、订单路由或个人实盘结果。

## 本地运行

要求：Python 3.11 或更高版本。

```bash
git clone https://github.com/mukhixiaoning122-create/futures-research-portfolio.git
cd futures-research-portfolio
python -m venv .venv
```

Windows PowerShell：

```powershell
.\.venv\Scripts\Activate.ps1
python -m pip install -e ".[dev,web]"
pytest -q
cta-showcase-web
```

Linux / macOS：

```bash
source .venv/bin/activate
python -m pip install -e ".[dev,web]"
pytest -q
cta-showcase-web
```

浏览器访问 `http://127.0.0.1:8000`。

如果只想在命令行查看同一份端到端演示报告：

```bash
cta-showcase
```
